Редактор драйвера

Этот диалог позволяет Вам изменять настройки драйвера.

Базовые параметры

Имя драйвера Имя драйвера, показанное в драйвер-менеджере, может быть использовано DBeaver'ом для именования скриптов, показа подсказок и т.д.
Категория Не обязательное поле. Некоторые базы данных имеют несколько различных драйверов, объединённых в одну группу. Категория - это имя такой группы. Большинство драйверов категории не имеют.
Имя класса Обязательное поле. Это имя Java класса, который реализует, собственно, драйвер. Вы можете найти имя класса в документации на драйвер или попробовать использовать кнопку "Найти класс" на закладке "Библиотеки драйвера" и выбрать имя класса из предложенного списка.
Пример адреса (URL) Не обязательное поле. Если это поле оставлено пустым, Вам придётся определить полный адрес (URL) базы данных. Гораздо удобнее, когда пример адреса (URL) определён, в этом случае мастер соединения потребует только отдельные параметры соединения (имя хоста, базы и т.д.).

Пример адреса (URL) - это JDBC-адрес базы данных, но вместо действительных параметров базы данных он состоит из "меток", которые будут заменены на реальные значения мастером соединения.

Список поддерживаемых меток в примере адреса:

  • {host} - имя хоста базы данных
  • {port} - порт базы данных
  • {database} - имя базы данных/схемы
  • {server} - имя сервера базы данных. Редко используется, как правило в "древних" базах данных (вроде Informix)
  • {folder} - директория базы данных. Обычно используется во встроенных базах данных для определения места расположения их файлов
  • {file} - то же, что {folder}, но определяет отдельный файл
  • {user}- имя пользователя
  • {password}- пароль пользователя

Необязательные метки заключаются в квадратные скобки

Типичный пример адреса (URL): jdbc:db2://{host}[:{port}]/{database}

Библиотеки драйвера

Список внешних библиотек драйвера. Обычно, библиотека - это jar или zip архивный файл, который содержит реализацию JDBC-драйвера. Вы можете определить целую директорию как источник классов драйвера вместо архивного файла.

Кнопка "Найти класс" под списком библиотек ищет все реализации JDBC-драйвера в указанных библиотеках. Вы можете выбрать определённое имя класса (если найдено несколько классов) и оно будет помещено в поле "Имя класса".

Свойства соединения

Эти настройки будут использованы всеми соединениями на основе этого драйвера. Однако, эти настройки могут быть переопределены в самом соединении.

Дополнительные параметры, используемые для настройки драйвера для специфического окружения или в случае особых требований. Список доступных параметров свой для каждого драйвера. Вы можете найти его в документации на драйвер или на закладке "Дополнительно" мастера соединений

Дополнительные параметры

Дополнительные параметры используются некоторыми драйверами для настройки специфичного поведения.

Список параметров, поддерживаемых обобщённым (Generic) драйвером:

Параметры завершения Некоторые встроенные базы данных требуют дополнительных параметров завершения в адресе (URL) при отсоединении. В противном случае, драйвер может заблокировать файлы базы данных до завершения процесса DBeaver'а. Если такой параметр определён, он будет добавлен к адресу (URL) базы данных при отсоединении. Обычно, параметры завершения выглядят приблизительно так: ;shutdown=true
Запрос получения активной базы данных SQL-запрос, который возвращает имя текущей базы данных/схемы. Используется DBeaver'ом для показа активной базы данных/схемы в интерфейсе.
Запрос, устанавливающий активную базу данных SQL-запрос, который устанавливает текущую базу данных/схему.

Лицензия

Лицензия драйвера. Если она присутствует, пользователь должен прочитать и принять её перед использованием драйвера.
